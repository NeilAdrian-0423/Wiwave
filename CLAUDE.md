# CLAUDE.md

## Project

**Wiwave** — a self-hosted, LAN-only voice chat. Vanilla JS client (no build step), Node + Express + ws server, WebRTC peer-to-peer audio, AudioWorklet with RNNoise + a customizable noise gate. Often hosted from an Android phone via Termux.

## Design Context

### Users

Families and housemates sharing the same Wi-Fi. **Not** tech enthusiasts — they're here to talk to the person in the next room, the kid upstairs, the partner cooking in the kitchen. They don't know what WebRTC or RNNoise are and shouldn't have to. They care about three things: easy, private, works.

Usage pattern: short bursts throughout the day. Phones, tablets, laptops. One-handed, often while doing something else. Lighting ranges from sunny breakfast table to dark bedroom at midnight.

### Brand Personality

**Calm. Private. Honest.**

A well-made tool that doesn't perform for attention. Trust comes from simplicity and restraint. The benchmark: an app a parent could open and use without being shown twice.

### Aesthetic Direction

- **Theme**: Follows the system (`prefers-color-scheme`). Light by day, dark by night.
- **Visual tone**: Between a well-designed reading app and a piece of handheld household hardware. Warm, not cold. Soft edges, generous spacing. Typography carries the load; decoration earns its place.
- **The visible machinery**: the audio meter. It's the app's proof-of-work. Everything else stays in the background.

**Anti-references**: enterprise dashboards, AI-template look (purple/cyan gradients, neon accents, gradient text, Inter-on-dark), Discord/Slack/Zoom, hacker-terminal maximalism.

### Design Principles

1. **Soft over sharp.** Warm neutrals, rounded corners that aren't cartoonish, low-contrast surfaces. Nothing clinical.
2. **Quiet confidence.** Trust type and spacing. If a decoration doesn't change what the user understands or feels, remove it.
3. **Respect the room.** Light mode = paper-white with warm neutrals. Dark mode = deep warm charcoal, not cold blue-black. Feels like a home, not an office.
4. **The meter is the machinery.** The DSP meter is the one place the tool visibly *works*. No other element competes with it for attention.
5. **One-thumb-friendly, always.** Phone-first. Thumb-sized targets. No pinch gestures, no tiny dismiss buttons, no scroll traps.
6. **Words chosen, not generated.** Every label is specific and human. Plain language is the brand.

## Conventions

- No build step — edits go straight into `public/*.js` and `public/*.css`.
- Hardcoded room list lives in `ROOMS` in `server.js`.
- DSP changes touch `public/worklet-processor.js` and the `setupDsp()` / UI wiring in `public/client.js`.
- Static assets served from `public/`; `/worklet.js` is dynamically concatenated by the server from `@jitsi/rnnoise-wasm` + `public/worklet-processor.js`.
- HTTPS is required (getUserMedia); self-signed cert generated by `npm run cert`.
- `.env` supplies `PORT` via Node's native `--env-file`.
